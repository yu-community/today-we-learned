## action-oriented, resource oriented

action-oriented의 특징

- API가 하는 일에 초점을 맞춰 정의
- 내부에서 구체적으로 어떤 일을 하는지 알기 어려움
- 관리가 어려움
- function과 유사한 형태를 띤다.
- 구현이 간단하고 쉬워 효율적
- 내부에서만 사용하는 api에 주로 사용

resource-oriented의 특징

- 스탠다드 메서드와 resource의 조합으로 정의
- 내부 로직을 유추할 수 있음
